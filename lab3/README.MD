Мы выбрали SpringREST, потому что он дает возможности к полному стеку для разработки веб-приложений и микросервисов. По нему в интернете много документации, уроков, примеров. Это полноценный фреймворк с большими возможностями по созданию REST API. Т.к.  SpringREST принадлежит к системе Spring, он позволяется использовать внедрение разных зависимостей.
